# ISCF-IA-04: Account Lifecycle Monitoring

## Control Name

**ISCF-IA-04: Account Lifecycle Monitoring**

This control ensures that identity lifecycle activities such as account
creation, modification, and deletion are monitored through audit logging
mechanisms.

------------------------------------------------------------------------

## Scope

This control applies to IAM components deployed within the OpenShift
environment responsible for managing user identities and authentication.

The workflow evaluates whether audit logs generated by IAM services
capture lifecycle events including:

-   Account creation
-   Account modification
-   Privilege changes
-   Account deactivation
-   Account deletion

The evaluation result is uploaded to Concert to establish the compliance
posture for the associated environment.

------------------------------------------------------------------------

## Expectation

The platform must generate and retain audit records for identity
lifecycle events.

Specifically:

-   Account creation events must be logged.
-   Account modification and privilege updates must be logged.
-   Account deactivation or deletion must be logged.
-   Logs must be continuously generated and accessible for monitoring.

Presence of these logs confirms that identity lifecycle activities are
being monitored and can be audited.

------------------------------------------------------------------------

## Prerequisites / Dependencies

The following resources must exist before executing the workflow:

-   A **Compliance Catalog** created in Concert.
-   A **Compliance Profile** created using the catalog.
-   An **Environment** defined in Concert.
-   IAM services deployed and generating audit logs.
-   Access to OpenShift cluster logs where IAM components run.

Required authentications:

-   **OpenShift authentication (`ocp_auth`)**
-   **Concert authentication (`concert_auth_key`)** (optional for
    Concert version 2.2.0 and above)

------------------------------------------------------------------------

## Assumptions

The workflow assumes:

-   IAM components are deployed in the specified namespace.
-   IAM services produce audit logs containing identity lifecycle
    events.
-   Pod logs are accessible through OpenShift authentication.
-   The logging pipeline is operational and not filtered or disabled.
-   Lifecycle events will appear within the retrieved log window.

------------------------------------------------------------------------

## Design

The workflow performs automated log analysis to validate lifecycle
monitoring.

High-level workflow steps:

1.  Retrieve IAM pod logs from the specified OpenShift namespace.
2.  Parse logs for identity lifecycle events.
3.  Identify events such as:
    -   Account creation
    -   Account modification
    -   Account deletion
4.  Validate that authentication logs exist to confirm the logging
    pipeline is active.
5.  Determine compliance status.
6.  Upload the compliance result to Concert using OSCAL-compliant
    assessment results.

------------------------------------------------------------------------

## Input

  -----------------------------------------------------------------------
  Input                  Description
  ---------------------- ------------------------------------------------
  profile_name           Compliance profile name in Concert

  environment            Target environment where the compliance posture
                         will be recorded

  ocp_namespace          Namespace where IAM components are deployed

  ocp_auth               OpenShift authentication used to retrieve IAM
                         pod logs

  concert_auth_key       Concert authentication key used to upload
                         assessment results
  -----------------------------------------------------------------------

Example defaults:

profile_name: profile_sovereign\
environment: production\
ocp_namespace: ibm-common-services

------------------------------------------------------------------------

## Output

The workflow produces a compliance status and uploads it to Concert.

### PASS

Lifecycle events such as account creation, modification, or deletion are
detected in IAM audit logs.

This confirms that account lifecycle monitoring is functioning
correctly.

### FAIL

No lifecycle monitoring events are detected in IAM logs.

This indicates that:

-   lifecycle monitoring may not be enabled, or
-   audit logs are not being generated or collected correctly.

The result is uploaded to Concert as an **OSCAL assessment result** for
the specified environment.
